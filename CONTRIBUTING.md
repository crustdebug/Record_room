# Contributing to Record Room

Thank you for your interest in contributing to Record Room! This document provides guidelines and instructions for contributing.

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/record-room.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes: `git commit -m 'Add some feature'`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## 📋 Code Style

- Use 2 spaces for indentation
- Use meaningful variable and function names
- Add comments for complex logic
- Follow existing code patterns in the project
- Keep functions small and focused

## 🧪 Testing

Before submitting a PR:
- Test all features you've modified
- Test on different browsers (Chrome, Firefox, Safari)
- Test responsive design on mobile devices
- Ensure no console errors

## 🐛 Bug Reports

When reporting bugs, please include:
- Clear description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser and OS information

## 💡 Feature Requests

We welcome feature requests! Please:
- Check if the feature already exists
- Clearly describe the feature and its benefits
- Provide examples or mockups if possible

## 📝 Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation if needed
- Add comments explaining complex changes
- Reference related issues in the PR description
- Be responsive to feedback and review comments

## 🎨 UI/UX Contributions

- Maintain the vintage/vinyl aesthetic
- Ensure accessibility (color contrast, keyboard navigation)
- Test on different screen sizes
- Keep animations smooth and purposeful

## 🔒 Security

If you discover a security vulnerability, please email us directly instead of opening a public issue.

## 📜 License

## 🙏 Thank You!

Your contributions make Record Room better for everyone. We appreciate your time and effort!
